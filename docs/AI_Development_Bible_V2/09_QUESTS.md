# 09 QUESTS

## Purpose

Quest standards and structure.

## Core Principles

-   Gameplay \> Exam
-   Teach → Practice → Reuse
-   Reward curiosity
-   Preserve player flow
-   Reuse existing assets first
-   Never remove working features without clear improvement

## Standards

-   Keep systems modular.
-   Prefer readability over complexity.
-   Balance fun, learning, and performance.
-   Refactor when beneficial.
-   Maintain consistency with Vision.

## AI Checklist

-   Read Vision.
-   Analyze existing implementation.
-   Improve rather than replace.
-   Test before finishing.
-   Document significant changes.
